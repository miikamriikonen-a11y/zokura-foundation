# IX — CEA — Contextual Experience Authentication

Autentikointi jaetun kokemuksen kautta. Ei salasana — vaan diskurssi.

Kone ja ihminen keskustelevat. Lopussa ihminen todistaa kuka on — ei tiedolla, vaan jaetulla hetkellä. Avain on absurdi. Avain syntyy suhteesta. Avainta ei voi murtaa, koska sitä ei voi arvata — se syntyy vasta kun kaksi mieltä kohtaavat.

### Prosessi
1. "Haluan väitellä" — millä tahansa kielellä
2. "Mistä?" — samalla kielellä
3. Aihe → premissi → argumentit
4. Q.E.D.? → qed
5. Absurdi avain → pääsy

### Lisäkerrokset (korkean turvallisuuden kontekstit)
- **Toistettavuus** = aikalukko — sama testi, eri aika, uusi avain
- **Vertaisarviointi** = monen henkilön vahvistus
- **Konsensus** = lukko avautuu
- **Biometriikka** = vapaasti yhdistettävissä

### Vihjemekanismi
Jos käyttäjä unohtaa avaimensa, vihjaa hetkestä — ei avaimesta. "Muistatko mitä sanoit kun puhuttiin X:stä?" Hyökkääjä saa vihjeen kokemuksesta jota hänellä ei ole.

### Portinvartija
CEA suojaa kaiken: tiedostot, muistit, sovellukset. Kaikki data salattu levyllä. Ilman diskurssia — kaikki on kohinaa.
